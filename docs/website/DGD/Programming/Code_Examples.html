Full Subsystems

The components in this section are significant pieces of functionality which may be added to your MUD. They are designed to handle a particular task within the MUD, and handle it entirely, or nearly so.

  • Dworkin, Par Winzell and Erwin Harte put together an Alpha implementation of SSH support for DGD. It works with Melville and the Kernel Library. Phantasmal should include SSHD support based on this package as of Phantasmal-0.017.
  • Erwin Harte's Example Error Manager: errord.c
  • Geir Harald Hansen's Example Object Manager: objectd.c , wiztool_addon.c
  • Jason Cone's Template for InterMUD3 for DGD: Mailing list message
  • John "West" McKenna's (buggy) sample object manager: Mailing list message
  • James Tait's InterMUD3 daemon code, imud3d.c.
  • Another ObjectD, this one by pete at ana dot sk. Here it is.
  • Bart van Leeuwen's partial reimplementation of the LPMUD dialect on top of DGD.

Snippets

  • Markus Tippman's LPC sqrt function: Mailing list message
  • Mikael Lind's example of calculating file size: file_size.c
  • Sampsa Ranta's one-line find-and-replace: implode(explode(find+str+find,find),replace_with)
  • Frank Schmidt's more consistent implode and explode: Mailing list message
  • Sampsa Ranta's color-code parsing example: From mailing list
  • John "West" McKenna's sprintf stuff: Source and documentation
  • Matt Jenkins' pager code, apparently for a Melville-based library.
  • Matt Jenkins' sendmail code, which requires modifying the DGD server, along with some reasons it's insecure.
  • Par Winzell's off-the-cuff implementation of shadows in DGD
  • David Jackson's original parser for commands, for use with Melville
  • Mikael Lind's simple time-of-day example.
  • Par Winzell's code to generate PNG files on the fly.
  • Mikael Lind's Mixed Toolkit (broken link, no longer available): Repository , Source , unix tarball (tgz file) or Windows zip file
